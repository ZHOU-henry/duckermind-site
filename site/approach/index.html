Method

Build with research discipline, not presentation theater.

The site language takes cues from world-class technology and AI organizations, but it is adapted for a multi-program portfolio: editorial restraint, visible systems thinking, dense but readable information, and clear hierarchy between parent brand and child products.

Three Critical Views

What makes this design direction defensible.

Brand View

The root site should feel like a command atlas, not a pitch deck.

The most common mistake is to make the parent domain louder than the projects. Here, the parent site is calm, strategic, and orienting. It gives shape to the portfolio rather than competing with the underlying products.

Systems View

Root pages and subdomains should be staged separately.

A subdomain does not need a dedicated application on day one. Redirects now, real apps later, keeps the architecture clean and the deployment burden low.

Research View

High-trust interfaces need visible methodology.

Research and intelligence products look more credible when the operating model, evidence discipline, and structure are visible from the first page instead of hidden in docs.

Deployment Logic

The lowest-friction rollout path.

1

Serve the main site from Caddy static files

This keeps the origin simple and easy to maintain behind Cloudflare.

2

Create five DNS records for the project subdomains

Each record points to the same server. Caddy decides whether the host serves content, redirects, or reverse proxies.

3

Start with redirects for all five child programs

The audience gets a clean domain system immediately without needing five separate deployments on the first day.

4

Promote each subdomain only when that product is ready

Agora can become the first true application host, with the other four staying as curated project pages until they mature.
