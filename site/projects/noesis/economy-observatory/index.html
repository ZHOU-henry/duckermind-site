Method Layer

How the prototype should be read

Live where possible

Use real-time public indicators

Energy, GitHub, and some infrastructure signals can refresh more often.

Slice where needed

Use periodic snapshots for slow variables

Talent, storage, and algorithm density often require quarterly or annual slices.

Composite by design

Some factors need inferred indices

The prototype distinguishes direct measures from weighted composite proxies.
